Classes

August 14, 2018
Python

There are billions of Google users worldwide with their personal information (name, age, email address, phone number, etc.) stored in the Google database. Let's say that I changed my phone number, and I am trying to update my information on my Google account. The Google database has to search for my Google account from billions of different ones, sort out the info for my phone number, and change its value. How does Google manage to organize this massive database? Because Google has an arranged structure of personal information for all individuals, using class would be an efficient way to do so. 

 

Class

Think class as the waffle pan and instances (objects produced by a particular class) as waffles. You have the same structure that you use to make different waffles. 

class Myclass ():
    x = 1

Here, you create a class called Myclass.

c = Myclass()
print(c.x)

 Then, you create an object that calls Myclass and print the value of x from the class.

 

To do complicated tasks, you can add various functions (waffle dough) to the class (waffle pan).

class Numbers ():
    a = 1
    def func (self):
        print(self.a)
    def func_2 (self):
        print(2*self.a)

Under the Numbers class, you have set a as one and added two functions: one that prints out the original a value and the other that prints out the value of a multiplied by 2.

N = Numbers ()
N.func ()
N.func_2()

1
2

When you run the two functions you have created for the Numbers class, it prints out 1 and 2 successfully.

You might be wondering why we use self in the functions. Because we are creating a new class and setting up our own "blueprint" for the computer, we need to refer to the current instance when we are producing one. You can set self as any other argument you want if you would like.

 


Let's look at another example.

class Student ():
    def __init__ (self, name, grade):
        self.name = name
        self.grade = grade
    def func (a):
        print("My name is ", a.name)
    def func_2 (b):
        print("I am in grade ", b.grade)

S = Student ("Sam", 12)
S.func ()
S.func_2()

__init__ function (self, parameter) is a code that automatically runs as soon as the instance is created. In other words, when you call S.func() and S.func_2(), the computer will execute the __init__ function without us having to indicate that. This is why we were able to get the output, although we did not specify where the name and grade arguments are coming from. 

Parameters are included in __init__ function, which requires you to input an accurate corresponding number of parameters you initially set when you call the function. Here, two parameters are used. This means that you must put exactly two in order to run the function successfully.

Q. Create a code that performs the foundational arithmetic operations in an atypical way using class. If the user asks the computer to add (+), it will subtract (-). If the user asks the computer to subtract, it will add. Same with multiplication and division: Multiplication sign (*) will operate division (/), and division sign will operate multiplication. The input will only be integers.

 

In this question, you have to use overloading operators that forcefully change a certain argument's attribute. For instance, the + sign can only be used to connect more than one string values in python. However, usage of underscores would enable the change of meaning in + sign into, in this case, subtraction. 

class strange():
    def __init__(self,num):    
        self.a = num
    def __add__(self, num):
        return self.a - num
    def __sub__(self,num):
        return self.a + num
    def __mul__(self,num):
        return self.a / num
    def __truediv__(self,num):
        return self.a * num

We create a class that works out the atypical arithmetic operation mentioned above with num as the parameter.

N = strange(5)

Here, the num is set as 5. 

print(N + 2)
print(N - 2)
print(N * 2)
print(N / 2)
